Client-Side Architecture

Background: For an insurance company we need to create an application where users can check their insurance details and pay premium. We need to create an MFE based client application for this company.

Requirement:
1.	Build two MFEs (of any part of the above application requirement) and a container application for those two (Choose any from React, Angular or Vue for these)
2.	No backend required; you can use any client storage solution for the data storage

Cross Cutting Concerns to be followed:
1.	Add a web worker for any task in one of the MFE
2.	Use webpack for bundling
3.	Use any CSS pre-processor
4.	Implement across MFE communication for any one use case of data sharing between the MFEs or the container application.

Deliverables:
1.	High level design document of the application (which also shows the above cross cutting concerns use cases)
2.	Link to code for your application (Container app + 2 MFES)
a.	 Also, share the document with steps to deploy it locally
3.	Bonus - deploy your application using any of the Heroku, Netlify, Azure etc. separately and share the link of deployed application.
